2020-11-11 Upgrade EKS cluster on PRODUCTION environment

Created by
Sarunas Krisciukaitis
Nov 11, 2020
2 min read2 people viewed

Prerequisites

Make sure you have kubectl v1.18 or higher.

1kubectl version --client 2Client Version: version.Info{Major:"1", Minor:"18", GitVersion:"v1.18.10", GitCommit:"62876fc6d93e891aa7fbe19771e6a6c03773b0f7", GitTreeState:"clean", BuildDate:"2020-10-15T01:52:24Z", GoVersion:"go1.13.15", Compiler:"gc", Platform:"darwin/amd64"}

Client Version: version.Info{Major:"1", Minor:"18", GitVersion:"v1.18.10", GitCommit:"62876fc6d93e891aa7fbe19771e6a6c03773b0f7", GitTreeState:"clean", BuildDate:"2020-10-15T01:52:24Z", GoVersion:"go1.13.15", Compiler:"gc", Platform:"darwin/amd64"}

If you have older version than 1.18, please update you kubectl (use brew or just download kubectl binary)

1

Upgrade EKS control-plane

Changed K8S version in TF code and applied new configurations.

2

Addons upgrade

https://docs.aws.amazon.com/eks/latest/userguide/update-cluster.html

Check kube-proxy version:

1kubectl get daemonset kube-proxy --namespace kube-system -o=jsonpath='{$.spec.template.spec.containers[:1].image}'

should be 1.18.8

Patch kube-proxy to new version

1kubectl set image daemonset.apps/kube-proxy \ 2-n kube-system kube-proxy=602401143452.dkr.ecr.us-west-2.amazonaws.com/eks/kube-proxy:v1.18.8-eksbuild.1

Check CoreDNS version:

1kubectl get deployment coredns --namespace kube-system -o=jsonpath='{$.spec.template.spec.containers[:1].image}'

Should be 1.6.6 - dont upgrade to 1.7.0 as its still buggy

Check CNI plugin version:

1kubectl describe daemonset aws-node --namespace kube-system | grep Image | cut -d "/" -f 2

Should be 1.7.5

Update cni plugin with following command if older:

1curl -o aws-k8s-cni.yaml https://raw.githubusercontent.com/aws/amazon-vpc-cni-k8s/v1.7.5/config/v1.7/aws-k8s-cni.yaml 2 3sed -i -e 's/us-west-2/eu-west-1/' aws-k8s-cni.yaml 4 5kubectl apply -f aws-k8s-cni.yaml

check version again

3

Upgrade cluster autoscaler

  1. Delete old cluster autoscaler which was installed with Helm2:

    1make delete-old env=production

  2. Install with newest one with Helm3:

    1make install env=production

3

Node upgrade

This script uses for node termination:

1#!/bin/bash 2 3nodes=($(kubectl get nodes --no-headers | grep v1.17 | awk {'printf $1"\t"'})) 4 5for i in "${nodes[@]}"; do 6 echo "Draining $i" 7 kubectl drain --delete-local-data --ignore-daemonsets=true --timeout=60s $i 8 sleep 3 9 id=$(aws ec2 describe-instances --filters Name=private-dns-name,Values=$i | jq -M -r '[.Reservations | .[] | .Instances | .[] | .InstanceId] | .[0]') 10 echo "Deregistering ALB $i" 11 aws elbv2 deregister-targets --target-group-arn arn:aws:elasticloadbalancing:eu-west-1:810856659284:targetgroup/eks2-prod-external/803f84176e266492 --targets Id=$id 12 sleep 300 13 echo "terminating $i - $id" 14 aws ec2 terminate-instances --instance-ids $id 15 sleep 30 16done

 

 

4

Update Confluence page

Update following Confluence page Used 3rd party core software.

E2E

In order to check that all K8S features are works as expected, we are using sonobuoy framework.

Download appropriate version of sonobuoy and run tests with:

1sonobuoy run --plugin e2e --kube-conformance-image-version v1.17.2

To check the status of tests run:

1sonobuoy status

To inspect E2E results

1sonobuoy retrieve 2 3sonobuoy results $archive

Clean sonobuoy after you finish with following command:

1sonobuoy delete --wait
